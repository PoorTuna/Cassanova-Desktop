## v0.1.1

Packaging-only release — no application changes since 0.1.0.

### Packaging

- **Windows portable build.** A single-file `.exe` is now published next to the NSIS installer. Run it without installing.
- **Snap published on GitHub releases.** The `.snap` no longer requires Snap Store credentials in CI; it ships alongside the AppImage and `.deb`.
- **Branded Windows + Linux icon.** Installers and window icons now use the Cassanova logo instead of the default Electron one.

### Build / CI

- Linux runner installs `libarchive-tools` and `snapcraft`, so the `pacman` and `snap` targets stop failing on a clean image.
- The icon source is pinned to `build/icon-square.png` to keep electron-builder from treating it as an `.icns` master.

### Known limitations

- **Binaries are unsigned.** Windows SmartScreen and macOS Gatekeeper will warn on first launch. Code-signing is on the roadmap but not in this release.
- **macOS icon falls back to the default.** The current logo asset is 294×298, below the 512×512 minimum for `.icns`. macOS builds still ship; they just use the stock Electron icon until a higher-resolution master is added.
